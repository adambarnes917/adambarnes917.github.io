Client Project Overview

Purpose:

  • Announce the creation of a club basketball team.

  • Provide information on how to join.

  • Share plans for practices, tryouts, and goals.

  • Connect with interested community members and potential sponsors.

Target Audience: This website is designed for students, coaches, fans and sponsors interested in being involved with the club basketball team.

Website Overview:

  • Home Page: Announcement and overview.

  • About Page: Team mission, goals, and vision.

  • Tryouts Page: Info on joining, registration form.

  • Gallery Page: Placeholder for future team photos and events.

  • Contact Page: Communication form for interest or sponsorship.

Tryouts

Purpose: Provide information about upcoming tryouts and allow users to register their interest in joining the team.

Audience: Prospective players interested in trying out for the club basketball team.

Page Content:

  • Tryout details: date(s), time(s), location, what to bring.
  • Eligibility and requirements (age groups, experience level, waiver info).
  • Registration form for players to sign up (name, email, phone, age, preferred position, emergency contact).

Does the user enter data? (Yes/No): Yes

Does data need validations? (Yes/No): Yes — required fields: name, email, age; email format validation; optional phone format checks and numeric / range validation for age.

Does the page contain buttons, hyperlinks or dropdowns? (Yes/No): Yes — form submission button, navigation links, and possibly dropdowns for position/age group.

What actions will happen on this page?: On submit, JavaScript/jQuery will validate inputs. If valid, the data will be sent via AJAX to a mock endpoint or stored locally (for demo). A confirmation message or modal popup will appear after successful submission. Error messages will show inline for missing/invalid fields.

Special Notes: Use client-side validation with helpful error text. Consider adding a CAPTCHA or honeypot field if spam is a concern. For the project demo, AJAX can post to a placeholder URL (or simulate success with setTimeout).

Gallery

Purpose: Showcase team photos, highlight events, and provide a visual connection with the community.

Audience: Fans, players, coaches, and community members interested in seeing team activities.

Page Content:

  • Image grid with placeholder photos to be replaced as events happen.
  • Captions or short descriptions for each image (date/event).
  • Optional embedded video highlights area (for future use).

Does the user enter data? (Yes/No): No

Does data need validations? (Yes/No): No

Does the page contain buttons, hyperlinks or dropdowns? (Yes/No): Yes — navigation links and clickable images that open in a lightbox.

What actions will happen on this page?: Clicking an image will open a lightbox gallery (e.g., a jQuery lightbox plugin) where users can navigate between images. Videos, when added, can open in a modal or embed player.

Special Notes: Use a jQuery lightbox or similar plugin for a polished viewing experience. Optimize images for web (use thumbnails for the grid and full-size for the lightbox).

Contact

Purpose: Allow interested players, coaches, and community members to reach out to the club for questions, support, or sponsorships.

Audience: Prospective players, coaches, sponsors, and community members wanting to connect with the club.

Page Content:

  • Contact form with: name, email, phone (optional), message, and subject (e.g., "Player Interest", "Coaching", "Sponsorship").
  • Club email address and social media links (Twitter/Instagram/Facebook as available).
  • Embedded map (iframe) showing the meeting / tryout location (placeholder until venue confirmed).

Does the user enter data? (Yes/No): Yes

Does data need validations? (Yes/No): Yes — all required fields must be filled; email must be validated for proper format; message should have a minimum length (optional).

Does the page contain buttons, hyperlinks or dropdowns? (Yes/No): Yes — form submit button, social links, and navigation links.

What actions will happen on this page?: When the form is submitted, JavaScript/jQuery will validate inputs then send them via AJAX to a contact endpoint (or simulate with a success message). On success, show a thank-you modal or inline confirmation and optionally clear the form. Map iframe is static and provides location context.

Special Notes: Use accessible form markup (labels tied to inputs). Consider server-side backup of contact messages in a real deployment; for the project, a simulated AJAX response is sufficient. Include privacy note if saving contact info.
